# Technical Integration Blueprint: GoHighLevel & VAPI

This document outlines the technical architecture for connecting the bilingual AI voice receptionist to GoHighLevel (GHL).

## 1. VAPI Assistant Configuration

### Tools Definition
The AI agents (English and Spanish) will use two primary tools to interact with GHL.

#### Tool: `book_appointment`
- **Purpose**: Sent when the caller agrees to an inspection time.
- **Parameters**:
    - `datetime`: ISO 8601 string of the requested appointment.
    - `address`: The property address.
- **Action**: This tool can either call a GHL API directly or be passed in the final webhook payload. *Recommendation: Pass as part of the post-call webhook to keep logic centralized in GHL.*

#### Tool: `transfer_to_spanish` (English Agent only)
- **Purpose**: Immediately transfers the call to the Spanish-language assistant.
- **Action**: VAPI "Transfer" action to the Spanish Assistant ID or a secondary phone number mapped to the Spanish Assistant.

## 2. Webhook Structure (VAPI -> GHL)

At the end of every call, VAPI sends a `call.ended` or `call.tool_call` webhook. We will configure a final tool or a server-side handler to POST to a GHL Inbound Webhook URL.

### Recommended Webhook Payload
```json
{
  "call_id": "{{call.id}}",
  "phone": "{{call.customer.number}}",
  "name": "{{call.customer.name}}",
  "address": "{{address}}",
  "living_area": "{{living_area}}",
  "roof_size_est": "{{roof_size_est}}",
  "material_preference": "{{material_preference}}",
  "quote_low": "{{quote_low}}",
  "quote_high": "{{quote_high}}",
  "urgency": "{{urgency}}",
  "insurance_claim": "{{insurance_claim}}",
  "appointment_requested": "{{appointment_time}}",
  "call_summary": "{{transcript_summary}}",
  "recording_url": "{{recording_url}}"
}
```

## 3. GoHighLevel Setup

### Custom Fields
The following custom fields must be created in the GHL sub-account:
- `Property Address` (Text)
- `Living Area SqFt` (Number)
- `Estimated Roof Size` (Number)
- `Material Preference` (Text)
- `Quote Range Low` (Currency)
- `Quote Range High` (Currency)
- `Lead Urgency` (Text)
- `Insurance Claim` (Text)
- `AI Call Summary` (Large Text)
- `Call Recording URL` (Text)

### Workflow Automation: "AI Lead Capture & Booking"

#### Step 1: Trigger
- **Inbound Webhook**: (Use the URL generated by GHL).

#### Step 2: Create/Update Contact
- Map `phone` to Contact Phone.
- Map `name` to Contact Name.
- Map all custom fields from the webhook to the GHL Custom Fields.

#### Step 3: Opportunity Creation
- **Pipeline**: Roofing Sales.
- **Stage**: New Lead / AI Qualified.
- **Value**: Use `Quote Range High` as the initial value.

#### Step 4: Conditional Logic - Appointment Booking
- **If** `appointment_requested` is NOT empty:
    - **Action**: "Book Appointment".
    - **Calendar**: Roof Inspection Calendar.
    - **Time**: Map from `appointment_requested`.
- **Else**:
    - **Action**: Add Tag `Follow-up Needed`.

#### Step 5: Communications
- **Action**: Send SMS to Contact.
    - *Template*: "Hi {{contact.first_name}}! This is [Company Name]. Your roof inspection is confirmed for {{appointment.start_time}} at {{contact.property_address}}. See you then!"
- **Action**: Internal Notification (SMS/Email to Owner).
    - *Template*: "New AI Lead: {{contact.name}} booked an inspection for {{contact.property_address}}. Est. Quote: {{contact.quote_range_low}} - {{contact.quote_range_high}}."

---

## 4. Error Handling & Fail-safes
- **No Appointment**: If the caller hangs up before booking, the webhook still fires. GHL creates the lead and tags it as `Incomplete Booking` for manual follow-up.
- **Bilingual Handoff**: The English agent must pass the `call_id` or `customer_phone` to the Spanish agent to ensure the two call segments can be merged or identified as the same lead in GHL.
